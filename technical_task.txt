ФАЗА 2. Деплой в контейнере / Render (~3–5 часов) — ЗАДАЧА 3
--------------------------------------------------------------------------------

  [ ] 2.1. Dockerfile
       - FROM node:18-alpine
       - COPY package*.json → npm ci --omit=dev
       - COPY . → CMD ["node", "server.js"]
       - EXPOSE 3000 (или PORT из env)
       Файл: Dockerfile

  [ ] 2.2. docker-compose.yml (локально)
       - Сервис app (build .)
       - Сервис db (postgres:15), volume, переменные POSTGRES_*
       - app зависит от db, DATABASE_URL для подключения
       Файл: docker-compose.yml

  [ ] 2.3. Старт приложения на Render
       - render.yaml или инструкция в README:
         * Web Service, Docker или Node
         * Build: npm install
         * Start: npm start
         * Env: DATABASE_URL (внешняя PostgreSQL Render), JWT_SECRET, SMTP_*, ADMIN_PASSWORD
       - Health check: GET /api/health
       Файлы: render.yaml (опционально), README

  [ ] 2.4. Инициализация БД на сервере
       - Один раз: npm run init-db или SQL database.sql вручную в Render Postgres
       - Не запускать DROP при каждом деплое в production (вынести seeds отдельно позже)

  [ ] 2.5. CORS и статика
       - cors origin из env ALLOWED_ORIGIN (URL Render)
       Файл: server.js

Результат фазы: сайт открывается по HTTPS на Render, БД подключена.

  [ ] 5.1. Зависимость и сервис
       - npm install nodemailer
       - services/mail.js: sendMail({ to, subject, html })
       - Настройки из .env: SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS, MAIL_FROM
       Файлы: services/mail.js, package.json

  [ ] 5.2. Когда отправлять письма
       | Событие                         | Кому                    |
       |---------------------------------|-------------------------|
       | Создана заявка с replacing_teacher | замещающему (teachers.email) |
       | Замещающий принял (accept)      | инициатору заявки       |
       | Замещающий отклонил (reject)    | инициатору              |
       | Заявка отменена                 | замещающему (если был назначен) |

       Вставить вызовы после успешных операций в server.js:
         POST /api/requests, PUT /api/requests/:id/respond, PUT .../cancel
